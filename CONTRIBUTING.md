# Contributing to HexaMLOps

Thank you for considering contributing to HexaMLOps! Here are some guidelines to help you get started.

## Reporting Issues

1. Check the [issue tracker](https://github.com/AmadeusITGroup/HexaMLOPs/issues) to see if the issue has already been reported.
2. If not, create a new issue and provide as much detail as possible.

## Submitting Changes

1. Fork the repository.
2. Create a new branch for your changes: `git checkout -b my-feature-branch`
3. Make your changes and commit them with clear commit messages.
4. Push your changes to your fork: `git push origin my-feature-branch`
5. Create a pull request to the main repository.

## Coding Standards

- Follow the existing code style.
- Write tests for new features or bug fixes.
- Ensure all tests pass before submitting a pull request.
